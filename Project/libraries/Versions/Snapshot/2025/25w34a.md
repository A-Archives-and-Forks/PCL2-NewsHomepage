## 新特性

- 服务器现在拥有了游戏内的行为准则

### 服务器行为准则

当连接到服务器时，现在会显示服务器行为准则屏幕。

- 游玩服务器必须接受服务器行为准则
- 会提供一个检查框，勾选后不会再显示行为准则
  - 如果行为准则更改，它还会再次显示


#### 服务器设置

- 一个新的布尔字段 `enable-code-of-conduct` 添加进了 server.properties
  - 如果将其设为 `true` 将会使服务器在与 `server.properties` 所在文件夹中的名为 `codeofconduct` 子文件夹中寻找行为准则文件
  - 在文件夹中的每一个文件名应符合 `<language_code>.txt` 格式
    - 上方的语言将会尝试匹配玩家的语言
    - 如果不存在匹配玩家语言的文件，会使用 `en_us` 的行为准则
    - 如果 `en_us` 也不存在，则会匹配任意文件


## 更改

- 现在更新了主菜单的全景图，展示了村庄中的铜傀儡

### 末地闪光

- 由闪光造成的音效现在有了更长的延迟
- "隐藏天空闪电效果" 无障碍设置被重命名为 "隐藏天空闪烁效果"，启用的同时也会隐藏末地闪光
  - 当闪光由设置禁用时，世界不会被闪光点亮，但是闪光源以及音效仍会显示与播放

### 方块、物品以及实体的微调

- 当铜傀儡受到隐身药水效果时，它的眼睛现在会隐藏
- 铜傀儡碰撞箱微调
  - 碰撞箱宽度现在为 0.6 格，以能够寻路通过门
  - 碰撞箱高度现在为 1.01 格，以不会被某些方块卡住

### 音效

- 铜傀儡以及铜箱子现在可以在 12 格外听到其发出的音效
- 分类音效现在被调整地更安静

## 数据包版本 84.0

- 在数据组件以及方块实体的玩家档案不再会自动解析
- 增加了 `fetchprofile` 命令以下载玩家档案

### 方块状态

- 从展示架上移除了方块状态 `align_items_to_bottom`
  - 它被替换为在展示架方块实体上有着相同名称的布尔字段
  - 在上个快照创建的有着这个属性的展示架将会重置为默认值（`false`）


### 命令

#### 增加了 `fetchprofile`

- 增加了从 Minecraft 服务器上获取玩家档案内容的命令
- 因为这个操作可能需要一些时间，它在工作时为异步进行
  - 当操作完成时结果会被打印，但是游戏会一直不间断地运行


- 在成功完成后，会显示以让用户复制的一条消息完全储存的 `minecraft:profile` 组件内容，或有着 `said` 组件的 `/give @s minecraft:player_head`
- 因为命令执行不会立即产生结果，这个指令返回值恒为 1

语法:

- `fetchprofile name <player name>` - 返回由玩家名（大小写不敏感）解析的档案
- `fetchprofile id <uuid>` - 返回由 id 解析的档案

### 数据组件

#### 更改了 `minecraft:profile`

- 之前这个组件的内容会在档案被解析时永久的改变，例如在从 Minecraft 服务时
- 为了避免这个意料之外的更改的问题，现在解析值不再存储，并且档案内容会一直保持不被更改
- 这意味着这个组件现在有两个行为：
  - 静态: 当其存在 `properties` 字段时，或 `name` 和 `id` 均设置或均未设置时
    - 在这个状态下，档案会直接渲染
    - 如果 `properties` 字段缺省，会使用默认皮肤（由 `id` 决定，与离线模式相同）
    - 只意味着在组件创建时皮肤变为只读
    - 这个选项适合装饰用途

  - 动态: 当没有 `name` 或 `id` 时（但不是两个都没有）
    - 当包含这个档案的物品在客户端渲染时，档案将会被解析为最近期的值
    - 这意味着这个物品将会一直渲染为目标玩家的当前皮肤
      - 注：如果玩家在游戏运行时更改了皮肤，其更改不可见直到客户端重启

    - 在解析时可能会造成延迟，在这时物品会渲染为默认皮肤
    - 当存在时，物品名称将会替换档案名称
      - 只有 `id` 的物品会展示默认名称
    - 具有动态档案的物品会用提示栏与静态物品区分

- 物品堆叠基于精确的内容，所以即使动态组件与静态组件解析结果完全一样，它们也不会堆叠
- 在以前的地图上的未被新版本加载的玩家的头通常将会转为动态内容

### 方块实体数据

#### 更改 `minecraft:skull`

- `profile` 字段被更改以匹配 `minecraft:profile` 组件
  - 这意味着其不可修改并只有在必要时才会解析

### 文本组件

- 粗体与斜体样式在绘制精灵图时会被忽略
