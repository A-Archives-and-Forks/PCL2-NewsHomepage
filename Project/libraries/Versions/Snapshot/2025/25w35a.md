### Minecraft 服务端管理协议（MSMP）

增加了一个服务器管理 API (JSON-RPC over WebSocket) 用于专用服务器

- 此 API 默认禁用，可以由服务器 properties 文件启用

  - `management-server-enabled=true`
  - `management-server-host=localhost`
  - `management-server-port=25585`

- 当启用时 API 可以通过 `ws://<management-server-host>:<management-server-port>` 访问
- 支持查询以及更新服务器状态(玩家、白名单、操作员、设置、游戏规则)
- 在状态更改时发送通知 (e.g. 如玩家加入、游戏规则更新)
- 调用 `{"id":1,"method":"rpc.discover"}` 返回一个包含了当前运行的服务器支持的方法以及通知的 API schema
- 数据生成器在报告输出文件夹中提供一个 API schema (`json-rpc-api-schema.json`) 反映了 `rpc.discover` 方法返回的内容
- API 遵守 [JSON-RPC 2.0 规范](https://www.jsonrpc.org/specification)
- 使用带有命名空间的方法，并且保留的命名空间为 `minecraft` （如：`minecraft:players`,` minecraft:allowlist/add`）以及 `notification` （如 `notification:players/joined`）
  - 可以使用自定义命名空间以及额外方法进行扩展
- 核心方法组: 玩家、白名单、操作员、服务器（保存、停止）、服务器设置、游戏规则
- 方法调用示例:
  - 请求:

```json
{
  "method": "minecraft:allowlist/add",
  "id": 1,
  "params": [ [{ "name": "jeb_" }] ]
}
```

- 响应:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": [{ "id": "853c80ef-3c37-49fd-aa49-938b674adae6", "name": "jeb_" }]
}
```

- 通知示例:

```json
{
  "jsonrpc": "2.0",
  "method": "notification:players/joined",
  "params": [{ "id": "853c80ef-3c37-49fd-aa49-938b674adae6", "name": "jeb_" }]
}
```

- 错误示例:
  - 请求: `{"method": "minecraft:foo/bar","id": 1}`
  - 响应:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "jsonrpc": "2.0",
    "id": 1,
    "error": {
      "code": -32601,
      "message": "Method not found",
      "data": "Method not found: minecraft:foo/bar"
    }
  }
}
```

- 错误以及错误代码遵守 [JSON-RPC 2.0 错误对象格式](https://www.jsonrpc.org/specification#error_object) *(译者注：并未完全遵守，见 [MC-301531](https://bugs.mojang.com/browse/MC/issues/MC-301531))*

## 更改

### UI

- 当添加一个新的服务器到服务器列表时，名称不再预填充。当名称留空时使用默认名称。
- 更多文本输入框现在支持使用鼠标拖拽选择文字

#### 鼠标指针

- 一些 UI 组件现在会更改鼠标指针的形状
  - 例如: 文本输入框、数字滑块、按钮、滚动栏
- 此外，字符标签可点击的部分（如链接）现在会将指针改为 “手” 状
- 这项特性现在可以通过在 "鼠标设置" 的 "允许光标变化" 菜单中更改

### 对方块、物品以及实体的微调

- 铜傀儡的碰撞箱微调
  - 现在碰撞箱是 0.98 格高，以可以寻路通过一格高的空间
- 在生成铜傀儡时创建的铜箱子现在与附近的其它铜箱子正常连接
- 铜傀儡现在使用鞍装备槽放置在其避雷针顶上的方块
  - 其头装备槽现在与正常的头装备槽一样，类似于人型生物

## 技术性更改

- 数据包版本现在为 85.0
- 资源包版本现在为 67.0

### 服务器 Properties

因使用了一些新游戏规则，一些服务器 properties 的被移除，以允许在服务器运行时更改这些设置。

- 因使用新游戏规则，移除了服务器 property `allow-nether`
- 因使用新游戏规则，移除了服务器 property `spawn-monster`
- 因使用新游戏规则，移除了服务器 property `enable-command-block`
- 因使用新游戏规则，移除了服务器 property `pvp`
- 增加了服务器 property `management-server-enabled` 以控制服务器是否启用 Minecraft 服务端管理协议
  - 默认为 `false`
- 增加了服务器 property `management-server-port` 以控制服务器 Minecraft 服务端管理协议的端口
  - 默认为 `25585`
- 增加了服务器 property `management-server-host` 以控制服务器 Minecraft 服务端管理协议的主机
  - 默认为 `localhost`
- 增加了服务器 property `status-heartbeat-interval` 以控制服务器发送通知到连接的客户端的时间间隔
  - 默认为 0，意味着被禁用

### 游戏规则

- 增加了 `pvp` 游戏规则控制，以控制玩家是否可以对其它玩家造成伤害
  - 默认为 `true`
- 增加了 `allowEnteringNetherUsingPortals` 游戏规则以控制下界是否可以通过传送门进入
  - 默认为 `true`
- 增加了 `spawnMonsters` 游戏规则以控制怪物是否自然生成
  - 默认为 `true`
- 增加了 `enableCommandBlocks` 游戏规则以控制命令方块是否启用
  - 默认为 `true`

### 文本渲染

#### `object` 文本组件

- 更新了该组件以支持显示其它非字符对象作为文本的一部分
- 新格式:

  - `object`: 对象的类型，字符串，值见下方
  - `<type-specific>`: 由对象类型决定的可选字段

- 与 `nbt` 文本组件相似，`object` 字段可以省略
  - 然而，为了检查错误，我们推荐提供

##### `atlas` 对象类型

- 与之前 `object` 组件有相同的行为，但是现在其分离为一个对象类型
- 与之前使用相同的格式

##### `player` 对象类型

- 渲染玩家的头
- 如果提供了部分档案，皮肤会被与 `minecraft:profile` 组件在玩家的头上以相同的方式解析
- 格式:
  - `player`: 玩家档案
    - `name`:玩家名称，可选字符串
    - `id`: 玩家 UUID，可选的 UUID
    - `properties`: 档案属性，可选的映射
  - `hat`: 控制是否渲染帽子层，布尔值，默认为 `false`

### 配方
